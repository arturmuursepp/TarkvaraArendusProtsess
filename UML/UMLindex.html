Mis on UML?

Unified Modeling Language (UML) / unifitseeritud modelleerimiskeel on diagrammide tähistus keel, mis võimaldab teil süsteeme ja tarkvara visualiseerida. See on tarkvaraarenduse üks populaarsemaid diagrammide koostamise vorme ja sai ISO standardiks 1997. aastal. Sellest ajast alates on tehtud mõned uuendused, et lisada täiendavaid UML-diagrammitüüpe ja toetada uusi tehnoloogiaid programmeerimisel.


UML diagrammide tüübid

Seal on mõned UML-diagrammid, mida kasutatakse enamikus olukordades tarkvaraarenduse või süsteemide modelleerimisel, sealhulgas IT-infrastruktuuris ja ärisüsteemides. Kolm parimat UML-diagrammi, millega peaksite rahul olema, hõlmavad Kasutusmalliskeeme, Klassiskeeme (kui olete programmeerija) ja Järgnevusskeeme.

  • Use case diagram / Kasutusmalliskeem
  • Class diagram / Klassiskeem
  • Sequence diagram / Järgnevusskeem
  • Activity diagram / Tegevusskeem
  • State diagram / Olekuskeem
  • Interaction overview diagram / Interaktsiooniülevaate skeem
  • Communication diagram / Suhtlusskeem

    • Lisa UML-diagrammid keerukate, suurte või spetsiifiliste süsteemide jaoks

    Ülejäänud kaheksat tüüpi UML-diagramme kasutatakse konkreetsetes olukordades, kõige sagedamini siis, kui töötate väga suurte süsteemidega, millel on suur hulk komponente.

    Timing diagram / Ajadiagramm

    See on eritüüpi Järgnevusskeem, mis jälgib objektide käitumist aja jooksul. Neid kasutatakse sagedamini manustatud süsteemide, näiteks pesumasina juhtimistarkvara programmeerimisel.

    Object diagram / Objektiskeem

    UML-i objektiskeemid on klassiskeemide erivorm. Nad visualiseerivad abstraktsemaid klassiskeeme nii, nagu oleksid need instantseeritud ja aitavad tarkvaraarendajatel kontrollida, kas nende abstraktsed struktuurid praktikas töötavad.

    Component diagram / Komponendiskeem

    Komponendiskeem näitab, kuidas keeruka tarkvarasüsteemi erinevad komponendid on omavahel seotud ja suhtlevad liideste abil. Neid ei kasutata lihtsamate või sirgjooneliste süsteemide jaoks.

    Deployment diagram / Evitusskeem

    See on veel üks spetsiaalne diagramm suurte ja keeruliste süsteemide jaoks, kus tarkvara on juurutatud mitmesse süsteemi. Tänapäeval on veebiteenustel, nagu Amazon Web Services (AWS) ja Google Cloud Platform (GCP), oma tähistused – need kujundid leiate nende draw.io kujundite teegist.

    Package diagram / Paketiskeem

    Paketiskeemid näitavad teie süsteemi sõltuvusi ja neid kasutatakse peamiselt siis, kui teil on suur koodibaas. Paketid võivad sisaldada kasutusjuhtude diagramme, et illustreerida süsteemi funktsionaalsust ja visualiseerida tarkvarasüsteemi kihilist arhitektuuri.

    Composite structure diagram / Liitstruktuuriskeem

    Kui teil on keeruline tarkvarasüsteem, millel on palju erinevaid kasutusjuhtumeid, visualiseerivad UML-i liitstruktuuriskeemid, mis programmi käivitamisel tegelikult juhtub. See esindab klassi sisemist struktuuri ja seda, kuidas see on seotud erinevate klassikomponentidega.

    Profile diagram / Profiiliskeem

    Profiiliskeemi abil saate stereotüüpide, märgistatud väärtuste ja piirangutega laiendada ja kohandada UML-i kasutamiseks kindlas domeenis (nt lennundus, tervishoid, finants) või platvormil (J2EE, .NET). Jällegi kasutatakse seda tüüpi diagramme tavaliselt äärmiselt suurte ja keeruliste süsteemide modelleerimiseks.
